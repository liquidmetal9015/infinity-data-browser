This Special Skill represents the user's ability to transform into a different type of Trooper with its own Attributes and Skills.

![transmuted0.jpg](https://assets.corvusbelli.net/wiki/transmuted0.jpg)

TRANSMUTATION

AUTOMATIC SKILL

REQUIREMENTS

*   The user must meet the condition indicated between brackets next to the Skill.

EFFECTS

*   Different types of Transmutation are characterized by the cause that triggers the Trooper's transformation, which forces or allows them to use their second [Unit Profile](/Unit_Profile "Unit Profile").
*   When the Trooper transforms, the Model corresponding to the first Unit Profile is replaced with the Model corresponding to the second Unit Profile. If there is no alternative Model, place a Transmuted (TRANSMUTED) Token (or the corresponding Token if there is a specific one) beside the user, indicating their new profile.
*   The [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute points of a Trooper possessing this Special Skill are shared among all of their Unit Profiles. So, any [Wounds](/Wounds "Wounds") received by one of the Unit Profiles also apply to the other Unit Profiles.
*   Similarly, [States](/States "States") are also shared by the different Unit Profiles, if applicable.

### Transmutation (X)

*   A Trooper with this Skill is deployed with their first Unit Profile.
*   The value listed between brackets in the Unit Profile after the name of this Skill specifies how many Wounds the Trooper must receive before changing to their second Unit Profile.
*   The Trooper and their initial Unit Profile **must** be replaced by the second Unit Profile—and the Model representing it—at the end of the Order in which the initial Unit Profile has received at least as many Wounds as the value between brackets after the name of this Skill.

### Transmutation (Escape System-X)

*   This works the same way as Transmutation (X), but when the Unit Profile and Model are replaced, an [Eclipse](/Eclipse "Eclipse") Special Ammo Circular Template must be placed, centered on the Model.
*   The number of Wounds required to change Unit Profiles is listed in the name of the Skill in brackets after "Escape System".

**Update** _PDF 5.2, October 2025_  

*   If the Trooper is in [Possessed](/Possessed "Possessed") State when they change to their second Unit Profile, their Possessed State is canceled.

### Transmutation (Hatching)

**Update** _PDF 5.2, October 2025_  

*   The Trooper with this Skill is deployed with its first Unit Profile, but using a Seed-Embryo Marker instead of a Model.

Show original text.

**Original**  

*   The Trooper with this Skill is deployed with its first Unit Profile, but using a Seed-Embryo Token instead of a Model.

*   If the Trooper has any Skill that allows it to deploy in Marker form, it uses that Marker instead of the Seed-Embryo Token. The [Camouflage Marker](/Camouflaged "Camouflaged") of a Troop with Transmutation (Hatching) has a [Silhouette](/Silhouette "Silhouette") (S) value of 2. If the Trooper is [Discovered](/Discover "Discover") or it reveals itself, the Marker is replaced with the Seed-Embryo Token.
*   Similarly, if the Trooper has any Deployment Skill with the [Airborne Deployment](/Airborne_Deployment "Airborne Deployment") Label, then it will deploy as a Seed-Embryo Token, and it will apply the rules of its Airborne Deployment Skill as usual.
*   The Trooper will change to its second Unit Profile and replace the Marker or the Seed-Embryo Token with the Model that corresponds to its Developed Form whenever:
    *   It declares a Skill with the [Movement Label](/Movement_Label "Movement Label") as the first Short Skill of its Order. When this happens, its second "Developed Form" Unit Profile is used for the entire [Order Sequence](/Order_Expenditure_Sequence "Order Expenditure Sequence").
    *   It performs a successful [Dodge](/Dodge "Dodge") [ARO](/ARO "ARO"). The Developed Form profile is applied before the Dodge move is performed.

### Transmutation (Auto)

*   The Trooper may deploy using any of their Unit Profiles.
*   **By performing any Skill with the [Movement Label](/Movement_Label "Movement Label")**, a Trooper with Transmutation (Auto) may switch between the different Unit Profiles listed in their Unit Profile. This change always happens at the beginning of their Movement, applying the new Unit Profile for the entire sequence of that Movement.
*   This will affect the distance the Trooper can move across the game table, depending on the [MOV](/MOV "MOV") Attribute of the chosen Unit Profile.

### Transmutation (WIP)

*   The Trooper with this Skill is deployed with its first Unit Profile.
*   When a Trooper activates, before declaring its [Order](/Trooper_Activation "Trooper Activation") or [ARO](/ARO "ARO") the Trooper may immediately change to its second Unit Profile by passing a [Normal](/Normal "Normal") [WIP](/WIP "WIP") Roll. When this happens, its second Unit Profile is applied for the rest of the current [Player Turn](/Player_Turn "Player Turn"). At the start of the [States Phase](/States_Phase "States Phase") the Trooper changes back to their first Unit Profile.
*   If the WIP Roll is failed, the Trooper continues to use its first Unit Profile, but may try again before each of their Orders or AROs.
*   If a MOD is listed after the Skill in the [Unit Profile](/Unit_Profile "Unit Profile"), that MOD applies to these WIP Rolls.

  

REMEMBER

**Game Sequence and Transmutation (X)/(Escape System-X):**

If the number of [Wounds](/Wounds "Wounds") received is equal to or higher than the value between brackets after the name of this Skill, the Unit Profiles must be changed after making the [Guts Roll](/Guts_Roll "Guts Roll").

  

Examples
--------

Transmutation (X)

A Trooper with Transmutation (2) receives three simultaneous hits. He makes his three [Saving Rolls](/Saving_Roll "Saving Roll") and fails two of them, so he receives 2 [Wounds](/Wounds "Wounds"). The Trooper changes to his second [Unit Profile](/Unit_Profile "Unit Profile") and replaces his current Model with the alternative one, placing his Wounds next to it. Thus, the Trooper will remain active on the table, but he still has 2 Wounds. If there is no alternative Model, the Transmuted Token is used to indicate the change of Unit Profile.

  

Transmutation (Escape System-X)

A Trooper takes multiple hits and receives 2 [Wounds](/Wounds "Wounds"). Since this is the number indicated in his Transmutation (Escape System-2) Skill, the player replaces the Model of the TAG with that of its Operator and places the [Eclipse](/Eclipse "Eclipse") Special Ammo Circular Template centered on the Operator.

If the Trooper is in [Stunned](/Stunned "Stunned") and [Immobilized-B](/Immobilized-B "Immobilized-B") States when the Unit Profile and the Model are changed, the Trooper retains both State Tokens.

  

Transmutation (WIP)

A Trooper with Transmutation (WIP-3) is activated in an [ARO](/ARO "ARO"). Before declaring the Skill it will perform, its player makes a [WIP](/WIP "WIP")\-3 Roll, and it is successful. Having passed the Roll, he may use the Trooper’s second [Unit Profile](/Unit_Profile "Unit Profile") for this ARO and for the rest of the [Player Turn](/Player_Turn "Player Turn").
