As a rule, the target of an Attack receives one Wound for each successful hit. This is represented by a Wound Token (WOUND) next to the target, with a numerical value to indicate how many Wounds the Trooper has received. If the target has received as many Wounds as the value of its [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute, place an [Unconscious](/Unconscious "Unconscious") State Token (UNCONSCIOUS) as well.

IMPORTANT

Certain types of Ammunition, Attacks, Equipment, Hacking Programs, etc. force the target to make more than one [Saving Roll](/Saving_Roll "Saving Roll") for each success in the Attack Roll (For example, [DA](/DA "DA") ammunition forces the target to perform 2 Saving Rolls for each successful Attack). Others inflict more than one Wound with each failed Saving Roll or cause a change in the state of the target.

In these cases, the special effects are explained in the rule or the [Weapon Chart](/Weapon_Chart "Weapon Chart").

  

Unconsciousness and Death
-------------------------

If a target receives **as many Wounds** as the value of its [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute, it enters [Unconscious State](/Unconscious_State "Unconscious State").

On the other hand, if the target receives **more Wounds** than the value of its VITA/STR Attribute, it enters [Dead State](/Dead_State "Dead State").

**[Doctor](/Doctor "Doctor")**

*   The target must have the [Vitality](/Vitality "Vitality") Attribute and be in [Unconscious State](/Unconscious_State "Unconscious State").

\[...\]

*   By passing a [Normal](/Normal "Normal") [WIP](/WIP "WIP") Roll, the target's Unconscious State is cancelled, by removing 1 single Wound from those it has received, unless a rule or Skill specified otherwise. If the WIP Roll fails, the target automatically enters the [Dead State](/Dead_State "Dead State") and is removed from the game table.

  
**[Engineer](/Engineer "Engineer")**

*   Engineer allows, by passing a [Normal](/Normal "Normal") [WIP](/WIP "WIP") Roll, removing a Wound from a target with the [Structure](/Structure "Structure") Attribute (STR). This cancels [Unconscious State](/Unconscious_State "Unconscious State") if applicable.

  
**[Regeneration](/Regeneration "Regeneration")**

*   During the States Phase, the user makes a [Normal](/Normal "Normal") [PH](/PH "PH") Roll:
    *   If the Roll is a success, the Trooper removes 1 Wound, cancelling the [Unconscious State](/Unconscious_State "Unconscious State") if applicable.
    *   If the Roll fails, the Trooper receives 1 additional Wound.

  
**[Protheion](/Protheion "Protheion")**

*   The target must have the [VITA](/VITA "VITA") Attribute.

\[...\]

*   For each Wound they inflict on the target of the [CC Attack](/CC_Attack "CC Attack"), or [Coup de Grâce](/CC_Attack#Coup_de_Gr.C3.A2ce "CC Attack") they perform, the user removes 1 Wound they have received. If the user hasn't received any Wounds, they increase their [VITA](/VITA "VITA") Attribute value by 1.

  
**[GizmoKit](/GizmoKit "GizmoKit")**

*   The target of a GizmoKit must be an [Allied](/Allied "Allied") Model with the [STR](/STR "STR") Attribute.

\[...\]

*   In both cases, by passing the PH Roll, the target removes a Wound. This cancels [Unconscious State](/Unconscious_State "Unconscious State") if applicable. If the PH Roll is failed, the target receives a Wound instead of removing it, entering Unconscious or [Dead State](/Dead_State "Dead State") if applicable.

  
**[MediKit](/MediKit "MediKit")**

*   A MediKit's target must be an [Allied](/Allied "Allied") Model with the [VITA](/VITA "VITA") Attribute and in [Unconscious State](/Unconscious_State "Unconscious State"), unless certain scenario rules say otherwise.

\[...\]

*   In both cases, by passing the PH Roll, the target may remove one Wound and cancel their [Unconscious State](/Unconscious_State "Unconscious State"). If the PH Roll is failed, the target automatically enters [Dead State](/Dead_State "Dead State") and is removed from the game table.

IMPORTANT

If a Trooper receives and removes [Wounds](/Wound "Wound") or [States](/States "States") in the same Order, the positive effect is always applied first, followed by the negative effect.

For example, if an [Engineer](/Engineer "Engineer") cancels a Trooper’s [IMM-A](/IMM-A "IMM-A") State at the same time that the same Trooper is hit by a Riotstopper, the Engineer’s WIP Roll to cancel the Trooper’s IMM-A State is made first, then the Trooper makes their PH-6 [Saving Roll](/Saving_Roll "Saving Roll") against the Riotstopper’s [PARA](/PARA "PARA") Special Ammunition. In this way, the original IMM-A Status could be canceled and a new one inflicted in the same Order.
