If a target receives **as many [Wounds](/Wounds "Wounds")** as the value of its [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute, it enters Unconscious State.

On the other hand, if the target receives **more Wounds** than the value of its VITA/STR Attribute, it enters [Dead State](/Dead_State "Dead State").

![eng-unconscious.jpg](https://assets.corvusbelli.net/wiki/eng-unconscious.jpg)

UNCONSCIOUS STATE

[Null](/Labels "Labels")

ACTIVATION

*   A Trooper or game element ("Trooper" from now on) receives as many [Wounds](/Wounds "Wounds") as their [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute value.

EFFECTS

*   **Restriction:** Troopers in this state cannot declare [Orders](/Trooper_Activation "Trooper Activation") or [AROs](/ARO "ARO").
*   Unconscious Troopers automatically enter **[Prone State](/Prone_State "Prone State")** unless they are a Troop Type unable to do so.
*   Troopers in this state do not contribute [Orders](/Orders "Orders") during the [Tactical Phase](/Tactical_Phase "Tactical Phase").
*   Automatic Special Skills and Automatic Equipment have no effect while in this state.
*   The Trooper does not count towards the Player's [Victory Points](/Victory_Points "Victory Points").

**Update** _PDF 5.2, October 2025_  

*   Troopers in Unconscious State still belong to their [Combat Group](/Combat_Group "Combat Group").

CANCELLATION

*   A [Wound](/Wound "Wound") is removed from the Trooper, leaving them with less Wounds than their [VITA](/VITA "VITA")/[STR](/STR "STR") Attribute value.
*   [Doctors](/Doctors "Doctors") can remove Wounds from Troopers with the VITA Attribute, while [Engineers](/Engineers "Engineers") can remove Wounds from Troopers with the STR Attribute. In both cases, the Wound is removed by performing the Doctor/Engineer Special Skill while in [Silhouette contact](/Silhouette_contact "Silhouette contact") with the affected Trooper, and passing a [Normal](/Normal "Normal") [WIP](/WIP "WIP") Roll (or the Roll specified by the Special Skill or Scenario).
*   Other Special Skills and pieces of Equipment allow the removal of Wounds as indicated in their rules.
*   Canceling **Unconscious** State automatically cancels **[Prone](/Prone "Prone")** State as well.

  

SEE ALSO

**[Dogged](/Dogged "Dogged")**

*   The Unconscious State of Troopers using the Dogged Special Skill has the following special rules:  
    \[...\]
*   Allows the Trooper to ignore the effects and [Null](/Null "Null") Label of Unconscious State, treating it as [Normal State](/Normal_State "Normal State") instead, but only for the rest of that Player Turn.

  
**[No Wound Incapacitation](/No_Wound_Incapacitation "No Wound Incapacitation")**

*   The Unconscious State of Troopers using the No Wound Incapacitation Special Skill has the following special rules:  
    \[...\]
*   Allows the Trooper to ignore the effects and [Null](/Null "Null") Label of Unconscious State, treating it as [Normal State](/Normal_State "Normal State") instead.

  
**[Shasvastii](/Shasvastii "Shasvastii")**

*   The Unconscious State of Troopers using the Shasvastii Special Skill (usually referred to as 'Shasvastii-Embryo') has the following special rules:  
    \[...\]
*   **During the game**, the Trooper is counted when calculating [Victory Points](/Victory_Points "Victory Points") for the [Retreat! situation](/Retreat_Situation "Retreat Situation"), and for any other rules or conditions required by the scenario.

  
**[Remote Presence](/Remote_Presence "Remote Presence")**

*   Troopers with Remote Presence and the [STR](/STR "STR") Attribute have two distinct levels of Unconscious State.
*   When the Trooper receives as many [Wounds](/Wounds "Wounds") as their STR Attribute value, they enter Unconscious State like any other Trooper. However, if the Trooper receives one Wound more than its STR value, they do not enter [Dead State](/Dead_State "Dead State") as usual, but instead enter a second level of Unconscious State, signified by a second Unconscious Token (UNCONSCIOUS)...

  
**[MediKit](/MediKit "MediKit")**

*   A MediKit's target must be an [Allied](/Allied "Allied") Model with the [VITA](/VITA "VITA") Attribute and in Unconscious State, unless certain scenario rules say otherwise.  
    \[...\]
*   In both cases, by passing the PH Roll, the target may remove one [Wound](/Wound "Wound") and cancel their Unconscious State. If the PH Roll is failed, the target automatically enters [Dead State](/Dead_State "Dead State") and is removed from the game table.

  
**[GizmoKit](/GizmoKit "GizmoKit")**

*   The target of a GizmoKit must be an [Allied](/Allied "Allied") Model with the [STR](/STR "STR") Attribute.  
    \[...\]
*   In both cases, by passing the PH Roll, the target removes a [Wound](/Wound "Wound"). This cancels Unconscious State if applicable. If the PH Roll is failed, the target receives a Wound instead of removing it, entering Unconscious or [Dead State](/Dead_State "Dead State") if applicable.

  
**[Regeneration](/Regeneration "Regeneration")**

*   During the States Phase, the user makes a [Normal](/Normal "Normal") [PH](/PH "PH") Roll:
    *   If the Roll is a success, the Trooper removes 1 [Wound](/Wound "Wound"), cancelling the Unconscious State if applicable.
    *   If the Roll fails, the Trooper receives 1 additional [Wound](/Wound "Wound").

  
**[Shock Ammunition](/Shock_Ammunition "Shock Ammunition")**

*   If the target has the **[VITA](/VITA "VITA") Attribute, with a value of 1**, any failed Saving Rolls against Shock Ammunition will also cancel any type of Unconscious State the target is in, or has entered during this Order, and the target passes directly to [Dead State](/Dead_State "Dead State").

  
**[CC Attack](/CC_Attack "CC Attack") > [Coup de Grâce](/Coup_de_Gr%C3%A2ce "Coup de Grâce")**

*   **Coup de Grâce**: When declaring the CC Attack, if the [Enemy](/Enemy "Enemy") [Target](/Target "Target") is in Unconscious State (including if it is represented by a Shasvastii-Embryo Token), then, without needing to perform a roll, the target automatically goes from Unconscious to [Dead State](/Dead_State "Dead State") without the possibility of making a [Saving Roll](/Saving_Roll "Saving Roll").  
    Coup de Grâce cannot be used against a Trooper who has activated the **[Dogged](/Dogged "Dogged")** or **[No Wound Incapacitation](/No_Wound_Incapacitation "No Wound Incapacitation")** Special Skill. Therefore, the CC Attack is resolved normally.
