ENHANCED REACTION

LONG SKILL

[Supportware, No Roll.](/Labels "Labels")

REQUIREMENTS

*   The target must be a [REM](/REM "REM") from the same [Army List](/Army_List "Army List") as the user and must be inside the user's [Hacking Area](/Hacking_Area "Hacking Area").

EFFECTS

*   Enhanced Reaction grants the target REM a [Burst](/Burst "Burst") value of 2 in [ARO](/ARO "ARO").
*   When the user activates this Program, which does not require a Roll, place a Supportware Enhanced Reaction Token (Sup: Enhanced Reaction) beside the user, and beside the REM.
*   Once activated, this Program's range covers the entire game table.

IMPORTANT

**SUPPORTWARE**

*   A Trooper can benefit from the effects of only one [Program](/Hacking_Program "Hacking Program") with the [Supportware Label](/Supportware_Label "Supportware Label") at a time.
*   Each [Hacker](/Hacker "Hacker") can only sustain one active Supportware Program at a time.
*   A Hacker can cancel an active Supportware Program by declaring another Program and expending the corresponding [Order](/Trooper_Activation "Trooper Activation") or [ARO](/ARO "ARO").
*   A Supportware Program is automatically cancelled if the target of the Supportware is targeted by a new Supportware Program, or if the Hacker running the Program enters [Isolated State](/Isolated_State "Isolated State") or any [Null](/Null "Null") States.

  

Example of Supportware Use and Cancellation

During their Active Turn, a Player that has two Troopers with [EVO Hacking Devices](/EVO_Hacking_Device "EVO Hacking Device") on the table, declares [Fairy Dust](/Fairy_Dust "Fairy Dust") with one of them and chooses [REMs](/REM "REM") as the [Troop Type](/Troop_Type "Troop Type") that will benefit from this Supportware.

For tactical reasons, they decide to spend the last Order on the other Trooper with an EVO Hacking Device to declare Enhanced Reaction on a REM located in the vanguard.

By designating a REM that was already benefitting from [Fairy Dust](/Fairy_Dust "Fairy Dust"), the new Supportware Program deactivates the Fairy Dust Program and the corresponding Token is removed from the table.

  

REMEMBER

A Supportware Program is automatically cancelled if the target of the Supportware is targeted by a new Supportware Program, or if the [Hacker](/Hacker "Hacker") running the Program enters [Isolated](/Isolated "Isolated") State or any [Null](/Null "Null") States.
