# Infinity Data Structure

This document describes the structure of the Infinity data files located in the `data` directory and provides a mapping of the files to their respective Factions/Armies.

## File Overview

### `metadata.json`
Contains definitions for the high-level shared data used across all armies.
- **factions**: List of all factions, sectorials, and valid IDs.
- **ammunitions**: Definitions of ammo types.
- **weapons**: Definitions of base weapons.
- **skills**: Definitions of special skills.
- **equipment**: Equipment definitions.

### Army Files (`*.json`)
Each JSON file in the `data` directory represents a specific Army or Sectorial.
The files appear to use two inconsistent naming conventions:
- `ID.json` (e.g., `301.json`)
- `new X.json` (e.g., `new 1.json`)

#### Army Data Structure
The root of an army file contains:
- `version`: The version string of the data.
- `units`: An array of Unit objects.

**Unit Object Structure:**
- `id`: Unique internal ID.
- `isc`: "International Standard Code" - effectively the unique name/key for the unit.
- `name`: Display name.
- `factions`: Array of Faction IDs that can take this unit.
- `profileGroups`: A list of profile groups (e.g., different loadouts or forms).
  - `profiles`: The actual stats (MOV, CC, BS, PH, WIP, ARM, BTS, W, S) and special rules for a specific profile.
  - `options`: The list of specific loadout options (Weapons, Skills, Equipment, Points, SWC).

## Identification Methodology
The mapping was generated by analyzing the `factions` array of every unit in each file.
- **Generic Armies** (e.g. PanOceania) typically have their primary Faction ID present in >50% of the units.
- **Sectorials** share many units with others, so their specific ID might not be present in every unit (e.g. Mercenaries), but it is the most frequent specific ID found in the file.
